Kubernetes 部署

一旦運行了 Kubernetes 叢集，就可以在其上部署容器化應用程式。 為此，你需要建立 Kubernetes Deployment 配置。Deployment 指揮 Kubernetes 如何建立和更新應用程式的例項。建立 Deployment 後，Kubernetes master 將應用程式例項排程到叢集中的各個節點上。

建立應用程式例項後，Kubernetes Deployment 控制器會持續監視這些例項。 如果託管例項的節點關閉或被刪除，則 Deployment 控制器會將該例項替換為叢集中另一個節點上的例項。 這提供了一種自我修復機制來解決機器故障維護問題。

在沒有 Kubernetes 這種編排系統之前，安裝指令碼通常用於啟動應用程式，但它們不允許從機器故障中恢復。透過建立應用程式例項並使它們在節點之間執行， Kubernetes Deployments 提供了一種與眾不同的應用程式管理方法。

部署你在 Kubernetes 上的第一個應用程式


你可以使用 Kubernetes 命令列介面 Kubectl 建立和管理 Deployment。Kubectl 使用 Kubernetes API 與叢集進行互動。在本單元中，你將學習建立在 Kubernetes 叢集上執行應用程式的 Deployment 所需的最常見的 Kubectl 命令。

建立 Deployment 時，你需要指定應用程式的容器映像以及要執行的副本數。你可以稍後透過更新 Deployment 來更改該資訊; 模組 5 和 6 討論瞭如何擴充套件和更新 Deployments。
