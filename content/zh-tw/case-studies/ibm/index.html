案例研究：
在 Kubernetes 上使用 Notary 和 TUF 建立映象信任服務

公司  IBM     位置  阿蒙克， 紐約     行業  雲計算

挑戰

IBM Cloud 提供公共、私有和混合雲功能，包括基於 OpenWhisk 的服務 （FaaS）、託管於 Kubernetes 和容器，以及 Cloud Foundry 服務 （PaaS） 的各種執行時。這些執行時與公司企業技術（如 MQ 和 DB2、其現代人工智慧 （AI） Watson 和資料分析服務）的強大功能相結合。IBM Cloud 使用者可以使用其目錄中 170 多個不同雲原生服務的功能，包括 IBM 的氣象公司 API 和資料服務等功能。在 2017 年後期，IBM 雲容器託管團隊希望構建映象信任服務。

解決方案

2018 年 2 月，這項新服務在 IBM 雲中公開發布。IBM 雲容器託管團隊的軟體開發者 Michael Hough 說，名為 Portieris 的映象信任服務完全基於 Cloud Native Computing Foundation (CNCF) 的開源專案 Notary。Portieris 是 Kubernetes 的准入控制器，用於強制執行適當的信任等級。使用者可以為每個 Kubernetes 名稱空間或在叢集級別建立映象安全策略，併為不同的映象強制實施不同級別的信任。Portieris 是 IBM 信任內容的關鍵部分，因為它使使用者能夠從 IKS 叢集中使用公司的 Notary。產品是 Notary 伺服器在 IBM 的雲中執行，然後 Portieris 在 IKS 叢集內執行。這使使用者能夠讓 IKS 叢集驗證他們載入容器的映象是否包含他們期望的內容，而 Portieris 是允許 IKS 叢集應用該驗證的原因。

影響

IBM 打算提供基於 Kubernetes 的容器服務和映象託管服務，目的是為其企業客戶提供完全安全的端到端平臺。Hough 說：“映象簽名是該產品的關鍵部分之一，我們的容器託管團隊將 Notary 視為在當前 Docker 和容器生態系統中實現該功能的實際方式。”該公司以前沒有提供映象簽名，Notary 是它用來實現該功能的工具。“我們有一個多租戶 Docker 託管服務，具有私有映象託管功能，” Hough 說。“ Docker 託管使用雜湊值來確保映象內容正確，並且資料在傳輸和靜態時都進行了加密。但它沒有提供任何保證誰推映象。我們使用 Notary 來允許使用者在其專用登錄檔名稱空間中籤名映象（如果他們願意的話）。”
“我們將 CNCF 視為雲原生開源的安全避難所，為成員專案（無論是原始供應商還是專案）提供穩定性、使用壽命和預期維護。”

- Michael Hough, IBM 容器託管團隊軟體開發人員

Docker 已經建立了 Notary 專案作為 The Update Framework (TUF) 的實現，TUF 的此實現為 Docker 內容信任提供了功能。

IBM 雲容器託管團隊的軟體開發者 Michael Hough 說：“在 TUF 和 Notary 對 CNCF 做出了貢獻後，我們發現它正在成為容器生態系統中映象簽名的實際標準。”

選擇 Notary 的關鍵原因是它已經與 IBM 的容器託管正在使用的現有身份驗證技術相容。TUF 的設計也是如此，它不要求託管團隊必須涉足金鑰管理業務。他說，這兩項都是“有吸引力的設計決定，證實了我們對 Notary 的選擇是正確的。”

在 IBM Cloud 中引入 Notary 功能以實現映象簽名，可提高 IBM 雲平臺的安全性，“我們預計這將包括簽署 IBM 官方映象以及預期的有安全需求的企業客戶，” Hough 說。與安全策略實現結合使用時，我們預計 CI/CD 管道中會更多地使用部署策略，以便根據映象簽名者對服務部署進行精細控制。 Hough 說，映象簽名的可用性“對於需要這種級別映象來源和安全性的客戶來說，是一個巨大的好處。”“藉助我們的 IBM 雲上的 Kubernetes 以及我們提供的許可控制器，它允許 IBM 服務以及 IBM 公共雲的客戶使用安全策略來控制服務部署。”
映象簽名是我們 Kubernetes 容器服務的關鍵部分之一，我們的容器託管團隊將 Notary 視為在當前 Docker 和容器生態系統中實現該功能的實際方式。

- Michael Hough, IBM 容器託管團隊軟體開發人員
現在，Notary 通常作為現有 IBM 雲容器託管的一個元件在 IBM 的公共雲中提供服務，因此它被部署為五個 IBM 雲區域中的高可用服務。此高可用性部署在五個區域中的每個區域中各有三個例項，實現負載均衡與故障轉移。Hough 說：“我們還將其部署到後端 IBM Cloudant 永續性儲存服務，並隨端到端 TLS 支援一起部署。”

IBM 團隊建立並開源了名為 Portieris 的 Kubernetes 准入控制器，該控制器使用 Notary 簽名信息與客戶定義的安全策略相結合，以控制將映象部署到叢集中。“我們希望透過使用我們的 Notary 服務來推動 Portieris 的使用，” Hough 說。

IBM 在建立和支援開源基礎（包括 CNCF）方面一直佔據主導地位。IBM 開放技術副總裁 Todd Moore 是現任 CNCF 董事會主席，許多 IBM 員工活躍於 CNCF 成員專案中。
“有新專案應對這些挑戰，包括在 CNCF 內。我們一定會饒有興趣地關注這些進步。我們發現 Notary 社群是一個積極友好的社群，對變化持開放態度，例如我們為持久儲存新增的 CouchDB 後端。”

- Michael Hough, IBM 容器託管團隊軟體開發人員
該公司已經使用的 CNCF 專案有 containerd，Envoy，Prometheus，gRPC，CNI，而且正在探索 SPIFFE 和 SPIRE 在未來的潛在可用性。

對於希望部署 Notary 或雲原生基礎架構的其他公司，Hough 有何建議？

“雖然對於雲原生基礎結構軟體的許多領域也是如此，但我們發現，高可用性、多區域的 Notary 部署需要紮實的實現來處理證書管理和輪換，”他說。“有新專案應對這些挑戰，包括在 CNCF 內。我們一定會饒有興趣地關注這些進步。我們發現 Notary 社群是一個積極友好的社群，對變化持開放態度，例如我們為持久儲存新增的 CouchDB 後端。”
