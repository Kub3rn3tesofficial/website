案例研究:
Ygrene: 使用原生云為金融行業帶來安全性和可擴充套件性

公司  Ygrene     位置  佩塔盧馬，加利福尼亞州     行業  清潔能源融資

挑戰

作為一家 PACE（清潔能源資產評估）融資公司，Ygrene 自2010年以來已經為超過10億的貸款提供資金。為了批准和處理這些貸款，“我們有很多正在聚合的資料來源，而且我們也有許多系統需要對這些資料進行改動，”Ygrene 開發經理 Austin Adams 說。該公司正在使用大量伺服器，“我們剛剛達到能夠垂直擴充套件它們的極限。我們有一個非常不穩定的系統，它變得不知所措，要求只是做後臺資料處理的實時。使用者看到的效能很差。我們需要一個解決方案，不需要我們對程式碼庫進行大量重構。作為一家金融公司，Ygrene 還需要確保他們安全地傳輸應用程式。”

解決方案

從 Engine Yard 和 Amazon Elastic Beanstalk 上遷移了應用後，Ygrene 團隊採用雲原生技術和實踐：使用Kubernetes來幫助垂直擴充套件和分配工作負載，使用 Notary 加入構建時間控制和獲取使用第三方依賴的可信賴 Docker 映象，使用Fluentd“掌握堆疊中的所有情況”，這些都執行在Amazon EC2 Spot上。

影響

以前，部署通常需要三到四個小時，而且每週或每兩週要把一些兩三個月工作量的任務在系統佔用低的時候進行部署。現在，他們用5分鐘來配置 Kubernetes，然後用一個小時進行整體部署與煙霧測試。Adams 說：“我們每週可以部署三到四次，只需一週或兩天的工作量。”“我們在工作周、白天的任意時間進行部署，甚至不需要停機。以前我們不得不請求企業批准，以關閉系統，因為即使在半夜，人們也可能正在訪問服務。現在，我們可以部署、上傳程式碼和遷移資料庫，而無需關閉系統。公司獲得新功能，而不必擔心某些業務會丟失或延遲。”此外，透過使用 kops 專案，Ygrene 現在可以用以前成本的十分之一使用 AWS EC2 Spot 執行其 Kubernetes 叢集。Adams 說，這些雲原生技術“改變了可擴充套件性、可觀察性和安全性（我們正在新增新的非常安全的資料來源）的遊戲。”“沒有 Kubernetes、Notary 和 Fluent，我們就無法告訴投資者和團隊成員，我們知道剛剛發生了什麼事情。”
“CNCF 專案正在幫助 Ygrene 確定整個 PACE 行業的安全性和可觀察性標準。我們是一個新興的金融企業，沒有這些專案，尤其是 Kubernetes，我們不可能成為今天的行業領導者。”

— Austin Adams, Ygrene 能源基金會開發經理

在不到十年的時間裡， Ygrene 就為可再生能源專案提供了超過10億美元的貸款。

PACE （清潔能源物業評估）融資公司開發經理 Austin Adams 表示：“我們抵押房屋或商業建築，用貸款來為任何可以節約電力、生產電力、節約用水或減少碳排放的專案提供資金支援。”

為了批准這些貸款，公司需要處理大量的承銷資料。Adams 說：“我們必須要驗證有關財產、公司或人員的問題，像這樣的工作數以千計。因此，我們有很多正在聚合的資料來源，並且我們也有大量系統需要實時對這些資料進行改動。”

到 2017 年，部署和可擴充套件性已成為痛點。該公司已經使用了大量伺服器，“我們剛剛達到能夠垂直擴充套件的極限，”他說。遷移到 AWS Elastic Beanstalk 並不能解決問題：“Scala 服務需要來自主 Ruby on Rails 服務和不同供應商提供的大量資料，因此他們要求從我們的 Ruby 服務以一種伺服器無法承受的速率獲取資訊。在 Elastic Beanstalk 上我們也有許多配置與應用不匹配。這僅僅是一個開始，我們也意識到我們這個系統非常不穩定。”
“CNCF 是眾多專案驚人的孵化器。現在，我們定期檢視其網頁，瞭解是否有任何新的、可敬的高質量專案可以應用到我們的系統中。它實際上已成為我們瞭解自身需要什麼樣的軟體以使我們的系統更加安全和具有可伸縮性的資訊中心。”

— Austin Adams, Ygrene 能源基金會開發經理
Adams 和其他團隊一起著手尋找一種具有變革性的解決方案，但“不需要我們對程式碼庫進行巨大的重構，”他說。作為一家金融公司，和可伸縮性一樣，Ygrene 需要更好的安全性。他們透過採用雲原生技術找到了答案：Kubernetes 幫助縱向擴充套件和分配工作負載，Notary 在各個級別實現可靠的安全性，Fluentd 來提供可觀察性。Adams 說：“ Kubernetes 是社群前進的方向，也是我們展望未來的證明。”

有了 Kubernetes，該團隊能夠快速將 Ygrene 應用程式用 Docker 容器化。“我們必須改變一些實現和程式碼，以及系統的構建方式，” Adams 說，“但我們已經能夠在一個月左右的時間內將主要系統引入 Kubernetes，然後在兩個月內投入生產。對於一家金融公司來說，這已經非常快了。”

怎麼樣？Adams 說，這些雲原生技術“改變了可擴充套件性、可觀察性和安全性（我們正在新增新的非常安全的資料來源）的遊戲。”“沒有 Kubernetes、Notary 和 Fluent，我們就無法告訴投資者和團隊成員，我們知道剛剛發生了什麼事情。”

Adams 說，尤其 Notary 簡直就是“天賜之物”。“我們要清楚，我們針對第三方依賴項的攻擊面較低，或者至少是託管的。因為我們使用 Notary 作為一個信任系統，我們也使用它作為區分，所以生產映象由 Notary 簽名，但一些開發映象就不簽署。這是為了確保未簽名映象無法進入生產叢集。我們已經在測試叢集中使用它，以使構建的應用更安全。”
“我們必須改變一些實現和程式碼，以及系統的構建方式，” Adams 說，“但我們已經能夠在一個月左右的時間內將主要系統引入 Kubernetes，然後在兩個月內投入生產。對於一家金融公司來說，這已經非常快了。”
透過使用 kops 專案，Ygrene 能夠用以前成本的十分之一從 Elastic Beanstalk 遷移到 AWS EC2 Spot 上執行其 Kubernetes 群集。Adams 說：“以前為了擴充套件，我們需要增加例項大小，導致高成本產出低價值。現在，藉助 Kubernetes 和 kops，我們能夠在具有多個例項組的 Spot 上水平縮放。”

這也幫助他們降低了在公共雲中執行所帶來的風險。“我們發現，基本上，如果我們能夠使用中斷可能性極低、無中斷歷史的 EC2 Spot 選擇例項類，並且我們願意付出足夠高的價格，我們幾乎可以得到和使用 Kubernetes 相同的保證，因為我們有足夠的節點，”軟體工程師 Zach Arnold 說，他領導了向 Kubernetes 的遷移。“現在，我們已經重新架構了應用程式的這些部分，使之不再位於同一臺伺服器上，我們可以推送到許多不同的伺服器，並實現更穩定的部署。”

因此，團隊現在可以在一天中的任何時間傳輸程式碼。阿諾德說：“以前這樣做是很危險的，因為它會拖慢整個貸款管理軟體。”“但現在，我們可以在白天安全部署。”

雲原生也影響了 Ygrene 的 50 多名開發人員和承包商的工作方式。Adams 和 Arnold 花了相當長的時間“教人們思考開箱即用的”，Arnold 說。“我們最終選擇了稱之為“航運四S”：安全、可靠、穩妥、快速。”（有關其安全部分的更多內容，請參閱他們關於"持續駭客攻擊"策略的文章。至於工程師，Adams 說，“他們已經能夠跟上軟體進步的步伐。我想一天結束時，開發人員會感覺更好，他們也會感覺與現代軟體開發社群的聯絡更加緊密。”

展望未來，Adams 很高興能探索更多的 CNCF 專案，包括 SPIFFE 和 SPIRE。“ CNCF 是眾多專案驚人的孵化器。現在，我們定期檢視其網頁，瞭解是否有任何新的、可敬的高質量專案可以應用到我們的系統中。它實際上已成為我們瞭解自身需要什麼樣的軟體以使我們的系統更加安全和具有可伸縮性的資訊中心。”
